# 반응성 테스트 체계 구축

## 목적

뷰포트별 반응성 테스트 항목을 정의하고, 각 항목의 pass/fail을 체계적으로 관리한다.
이슈 발견 시 스크린샷을 `temp/screenshots/`에 저장하여 Notion 보고에 활용.

## 요구사항

- 뷰포트 크기별(320/375/768/1920px) 테스트 항목 정의
- 기능 테스트 항목 정의 (버튼, 토글, 추첨, 이력 등)
- 각 항목의 pass/fail 상태 추적
- Playwright MCP 기반 E2E 자동화

## 구현 범위

### 포함
- 반응성 테스트 체크리스트 작성 (TESTPLAN.md)
- 뷰포트별 레이아웃 검증 항목
- 기능별 E2E 검증 항목
- 테스트 실행 및 결과 기록

### 제외
- 테스트 자동화 스크립트 작성 (이번 스코프 아님)
- 성능(LCP, CLS 등) 측정

## 의존성

- 선행 workspace: 없음
- 전제: `npm run serve` (port 8080) 실행 상태
- 도구: Playwright MCP (`--browser msedge`)

## 설계 방향

이번 세션에서 수행한 테스트를 기반으로 항목을 체계화:

### 기능 테스트 (뷰포트 무관)
1. 추첨하기 (1세트/다세트)
2. 세트 수 변경
3. 다크 모드 전환/복원
4. 번호 제외 (선택/해제/초기화)
5. 제외 번호 반영 추첨
6. 이력 보기/숨기기
7. 이력 전체 삭제 (confirm/alert)
8. 복사 버튼

### 뷰포트별 레이아웃 테스트
| 뷰포트 | 크기 | 검증 포인트 |
|--------|------|------------|
| 극소형 모바일 | 320x568 | 헤더 1줄 + 잘림 없음, 제외 그리드 5열, 볼 2줄 배치 |
| 모바일 | 375x667 | 헤더 1줄, 제외 그리드 5열, 볼 2줄 배치 |
| 태블릿 | 768x1024 | 중앙 정렬, 제외 그리드 9열, 볼 1줄 배치 |
| 데스크톱 | 1920x1080 | 중앙 정렬, 제외 그리드 9열, 볼 1줄 배치 |
