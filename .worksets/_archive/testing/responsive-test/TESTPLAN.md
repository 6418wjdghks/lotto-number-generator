## 테스트 방법

- **도구**: Playwright MCP (`--browser msedge`)
- **서버**: `npm run serve` (localhost:8080)
- **절차**: 각 뷰포트에서 `browser_resize` → 페이지 로드 → snapshot/screenshot → 검증
- **overflow 검증**: `browser_evaluate` → `document.documentElement.scrollWidth <= document.documentElement.clientWidth`
- **CSS 속성 검증**: `browser_evaluate` → `getComputedStyle(element).property`
- **스크린샷 정책**: `temp/screenshots/`에 저장, 이슈 없으면 삭제, 이슈 있으면 Notion 보고

### 테스트 실행 순서

1. A 그룹 (기능 테스트) — 기본 뷰포트 375x667에서 실행
2. B1~B2 (모바일) → B3~B4 (경계값) → B5~B6 (태블릿/데스크톱) 순서
3. C 그룹 (교차 테스트) — 마지막

### 테스트 간 의존성

- B 그룹 실행 전 A1~A3 통과 필수 (추첨 결과가 있어야 볼 배치 확인 가능)
